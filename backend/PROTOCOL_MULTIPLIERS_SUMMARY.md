# Protocol Percentile Multiplier Scales

This document outlines the proposed multiplier scales for each protocol based on percentile rankings.

## How It Works

- Each protocol has a base point value (from `PROTOCOL_POINTS`)
- Multipliers are applied based on the user's percentile rank within that protocol
- Percentile thresholds: 99.99 (top 0.01%), 99.9 (top 0.1%), 99 (top 1%), 97.5 (top 2.5%), 95 (top 5%), 90 (top 10%), 75 (top 25%), 50 (top 50%), 25 (top 75%)
- Final points = base points × multiplier

## Notes

- If a user has no percentile data for a protocol, the multiplier defaults to **1x** (no change)
- Multipliers are applied before the networth and protocol count multipliers
- The percentile thresholds are checked from highest to lowest, so the highest applicable multiplier is used
